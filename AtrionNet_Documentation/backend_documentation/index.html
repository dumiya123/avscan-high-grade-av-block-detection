Backend Architecture

The backend is built as a high-performance RESTful API using FastAPI, serving as the bridge between the React frontend and the PyTorch ML Inference engine.

Technologies Used

FastAPI (Asynchronous) Uvicorn (ASGI) FPDF / ReportLab Pydantic (Data Models)

API Endpoints Specification

GET /health

Checks the status of the server and confirms the ML model is correctly loaded into GPU memory.

POST /preview

Lightweight endpoint for initial signal visualization. Returns a subsampled version of the ECG for frontend plotting without full AI analysis.

POST /analyze

The primary inference endpoint. Triggers the full Attentional-AtrionNet v5.0 pipeline. Generates AI heatmaps, diagnostic labels, and clinical PDF reports.

GET /report/{report_id}

Serves dynamically generated medical PDF reports for clinical download.

Code & File Purpose

backend/main.py

The core router and orchestration layer. Managed CORS, file uploads, and directs data to the `AVBlockPredictor` in the ML layer. This file ensures that GPU memory is managed efficiently by initializing the model on startup.

backend/reports/

Directory for persistent storage of generated PDF medical reports. These are accessible via unique report identifiers generated during analysis.

backend/uploads/

Temporary buffer for incoming physiological signals (.npy files). Files are automatically purged after successful analysis to maintain HIPAA-compliant data handling practices.

System Analysis Workflow

The backend follows a strict **"Load-Process-Generate"** pattern:

  1. Receive binary file (.npy) via multipart form data.
  2. Perform signal integrity check (Size & Shape validation).
  3. Execute synchronous AI inference on GPU.
  4. Post-process results (Scaling heatmaps, calculating heart rate).
  5. Generate PDF Report and return structured JSON to Frontend.
