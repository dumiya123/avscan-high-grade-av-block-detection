                         PART 1: UNDERSTANDING THE DATA TIER
================================================================================

WHAT IS THE DATA TIER?

The Data Tier is where all data is stored and managed. Think of it as the 
"warehouse" and "library" of your system.

It has 4 main storage areas:
1. File System Storage (for ECG files and reports)
2. Model Storage (for AI models)
3. Cache Layer (for fast access)
4. Session Storage (for tracking requests)

================================================================================
                    PART 7: HOW TO DRAW IN DRAW.IO
================================================================================

DRAWING A DATA TIER MODULE:
------------------------------------------------------------------------------

1. Select "Rectangle" tool

2. Draw a large rectangle (280px √ó 200px)

3. Format the rectangle:
   - Right panel ‚Üí Fill Color: Choose from palette
     (Blue for File System, Purple for Model, Orange for Cache, Green for Session)
   - Border: 3px
   - Border Color: Darker shade of fill color

4. Add title:
   - Double-click rectangle
   - Type module name
   - Format: Bold, 14pt, centered at top

5. Draw internal components:
   - Draw smaller rectangles inside (240px √ó 40px)
   - Fill: White
   - Border: 1px, dark color
   - Add component names


DRAWING INTERNAL DATA TIER ARROWS:
------------------------------------------------------------------------------

1. Click "Connector" tool

2. Click source component, then destination component

3. Format arrow:
   - Width: 2pt
   - Color: Dark Gray (#666666)
   - End Arrow: Classic
   - Pattern: Solid

4. Add label by double-clicking arrow


DRAWING TIER CONNECTION ARROWS (THE BRIDGE):
------------------------------------------------------------------------------

1. Click "Connector" tool

2. Click component in Logic Tier (above)

3. Click component in Data Tier (below)

4. Format arrow:
   - Width: 4pt (THICKER!)
   - Color: Blue (#2196F3)
   - End Arrow: Classic, Size 8
   - Pattern: Solid (for downward) or Dashed (for upward return)

5. For bidirectional arrows:
   - Draw the arrow
   - In right panel, set "Start Arrow" to Classic also
   - Now it has arrows on both ends!

6. Add label:
   - Double-click arrow
   - Type label
   - Format: Bold, 11pt, Blue color


MAKING ARROWS LOOK BEAUTIFUL:
------------------------------------------------------------------------------

1. Avoid crossing arrows:
   - Click arrow
   - Drag the middle to create curves
   - Use waypoints for complex paths

2. Align connection points:
   - Arrows from Logic Tier should come from bottom of boxes
   - Arrows to Data Tier should go to top of boxes
   - This creates a clean vertical flow

3. Consistent spacing:
   - Keep similar arrows parallel
   - Maintain equal spacing between parallel arrows

4. Label placement:
   - Place labels on the side of arrows
   - Avoid overlapping labels


================================================================================
                    PART 8: VISUAL ENHANCEMENT TIPS
================================================================================

MAKING THE DIAGRAM IMPRESSIVE:

1. COLOR CODING:
   - File System: Blue (represents files and documents)
   - Model Storage: Purple (represents AI/intelligence)
   - Cache Layer: Orange (represents speed/performance)
   - Session Storage: Green (represents active/live data)
   - Connection arrows: Blue (represents data flow)

2. VISUAL HIERARCHY:
   - Tier connection arrows (4pt) are thickest = most important
   - Cross-module arrows (2.5pt) are medium = important
   - Internal arrows (2pt) are thinnest = details

3. FLOW DIRECTION:
   - Downward arrows (solid) = Data going to storage
   - Upward arrows (dashed) = Data returning from storage
   - Bidirectional arrows = Active communication

4. GROUPING:
   - Use background rectangles to group related modules
   - Add a light gray background behind entire Data Tier
   - This separates it visually from Logic Tier

5. ANNOTATIONS:
   - Add a text box: "DATA TIER" at the top
   - Add a text box: "LOGIC TIER" above your existing diagram
   - Use large, bold font (18pt)
